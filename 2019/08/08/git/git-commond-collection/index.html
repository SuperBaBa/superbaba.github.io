0. 日常操作

  1. 拉取项目(默认master)
    git clonse 地址

  2. 抓取远程仓库代码到本地
    git pull

  3. 推送本地代码到仓库
    git push

  4. 查看所有分支
    git branch -a

  5. 切换分支
    git checkout

  6. 合并指定分支到当前分支
    git merge origin/

  7. 从当前本地分支创建新的本地分支,并切换到新分支
    git checkout -b

  8. 提交本地分支到远程仓库
    git push origin

  9. 仓库指针指向设置
    git branch –set-upstream-to=origin/

  10. 删除本地分支
    git branch -D

  11. 删除远程分支
    git push origin –delete

  12. 打标签
    git tag v1.0.0.1 –m “版本说明” —-标签版本号，说明
    git push origin v1.0.0.1 ——–推送至服务端

  13. 生成一个可供发布的压缩包
    git archive

  14. Git 仓库地址修改方法
    git remote set-url origin [NEW_URL]

  • Workspace：工作区
  • Index / Stage：暂存区
  • Repository：仓库区（或本地仓库）
  • Remote：远程仓库
